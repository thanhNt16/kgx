6.1 Index Performance (build_full_bulk / GraphBuffer)

Measured with KGX_LLM=mock KGX_EMBED=mock kg index --full. The GraphBuffer Rust port mirrors codebase-memory-mcp's C bulk-write architecture: drop indexes, synchronous=OFF, 64MB cache, single-transaction bulk INSERT, then recreate indexes + vec0/FTS post-processing.

Without edges (10k notes)

With edges (10k nodes + 30k edges) — instrumented breakdown

Total (cold, incl. ~32s startup)~38.2s—
Indexing only (warm, excl. startup)6.2s0.62ms/note

At 10k scale the indexing is I/O bound on vec0 (90% of time) because each of 10k notes gets an individual DELETE + INSERT INTO notes_vec. Bulk notes+edges INSERT takes only 25ms combined. A single-transaction vec0 batch write would reduce the 5.6s to ~10-50ms.

7. Graph Visualization (3D WebGL)

A full Three.js + React Three Fiber graph renderer has been ported from codebase-memory-mcp. Architecture:

ComponentDetail
RenderingReact Three Fiber (Three.js 0.183, drei 10.7, react-postprocessing 3.0)
Node displayInstancedMesh for <75k nodes, per-type color mapping, adaptive density auto-scaling
Edge displayLineSegments with typed color lookup, directional arrows
Post-processingBloom + glow via @react-three/postprocessing
NavigationOrbitControls with auto-rotate, pinch-to-zoom, focus animation
ControlsDepth slider, entity-type filter, background toggle, fullscreen + screenshot
Data formatGraphJSON — {nodes: [{id, label, entity_type}], edges: [{source, target, type}]}

Data flow: kg graph --format json → GraphJSON → R3F scene → NodeCloud (sphere InstancedMesh) + EdgeLines (LineSegments) + NodeLabels (CSS2DRenderer labels) + bloom/environment → user interaction via ControlTab.
