Quick Start — 5 Minutes

Install, wire your AI agent, and run your first /kgx: skill.

1. Install kg

curl ... install.sh | bash → kg in ~/.local/bin

2. Register MCP

claude mcp add --transport stdio kgx -- kg mcp-server --transport stdio

3. Scaffold vault

/kgx:init code → raw/, notes/facts/, notes/entities/

4. Work with content

/kgx:capture → /kgx:extract → /kgx:index → /kgx:ask

End-to-End Workflow

Each /kgx:<verb> runs the right CLI commands and formats results.

0

/kgx:init code

Scaffold a vault with the code/engineering template. Creates CLAUDE.md, index.md, raw/, notes/ with type subdirectories.

1

/kgx:capture

Save source material immutably. SHA-256 dedup. Raw files are never edited after capture.

2

/kgx:extract --source <ID>

Extract atomic facts, entities, decisions. Each note carries source: + recorded_at: provenance.

3

/kgx:index

Build SQLite brain — vector search (384-dim), BM25/FTS5, PageRank, Leiden communities. Deterministic, safe to run anytime.

4

/kgx:ask "question"

Hybrid vector + keyword + graph retrieval. Answers include cited note IDs. 89% token reduction vs paste-all.

5

/kgx:recall --entity "Name"

Entity neighborhood via 2-hop PageRank. No LLM call, pure graph lookup. Surfaces notes from 2 sprints ago.

6

/kgx:dream

7-pass consolidation: dedup, contradiction, supersession, stale archival, link repair. 2ms on 17 notes.

7

/kgx:status

Vault health at a glance. Nodes, edges, orphans, pending diffs, last index/dream timestamps.

8

kg graph --format html

Interactive 3D graph. Three.js with bloom, force-directed layout, orbit controls, node filtering.

Architecture

Markdown is canonical. The SQLite brain is a derived, git-ignored index — always rebuildable. 18 crates in a Rust workspace.

Markdown Vault (git canonical) raw/ ← immutable captured sources notes/ facts/ ← atomic claims (type: fact) entities/ ← people, systems, concepts decisions/ ← ADRs moc/ ← maps of content questions/ ← open gaps .kg/ ← derived, git-ignored brain.sqlite ← nodes, edges, FTS5, vectors, PPR, communities meta.json ← last-run timestamps metrics.log ← per-command token JSONL Brain Layers Vector KNN (sqlite-vec, 384-dim MiniLM) BM25 / FTS5 (porter-stemmed) Tags-in-LIKE substring search Tag-frequency-weighted expansion Entity NER → 1-hop graph expansion Leiden community summaries (GraphRAG) Personalized PageRank (HippoRAG) ↓ Reciprocal Rank Fusion Hybrid ranked results ↓ kg ask / kg recall / kg search / MCP tools

Lessons Learned (36-Sprint)

  1. Frontmatter is strict. kg enforces exact variant enums — prevents data rot.
  2. BM25 + PageRank is fast. 0.015s average across 653 nodes. No API key needed.
  3. Orphans accumulate. 73 of 363 notes (20%). Periodic link maintenance needed.
  4. Batch capture scales linearly. 290 files in 15s, no optimization needed.
  5. Pre-indexed search beats grep ~30x. Gap widens as vault grows.
  6. Safe name sanitization must handle / in addition to spaces.
