What is KGX?

A single kg binary that turns any Markdown vault into a hybrid vector + graph + keyword brain — queryable by you, your AI tools, and your team.

📥

Capture

Ingest any source (file or stdin) into an immutable raw/ directory with full provenance.

⚗️

Extract

AI extracts atomic facts, entities, and decisions from raw sources — each linked back to its origin note.

🧠

Index

Builds a SQLite "brain" with vector search (384-dim), BM25/FTS5, PageRank, and Leiden community detection. 17 nodes, 28 edges, 17 embedded — 1ms.

💬

Ask

Hybrid Q&A with citations. 371 input tokens measured (vs ~3,400 for paste-all on a 17-note vault).

🌙

Dream

7 nightly consolidation passes: dedup, contradiction, supersession, staleness, link repair. 14 diffs staged, 2ms.

🔌

MCP Server

JSON-RPC 2.0 over stdio. Claude Code, Codex, and Cursor connect directly to your brain. 6 tools. Errors return JSON-RPC envelopes — server never crashes.

Architecture

Markdown is the canonical source of truth. The SQLite brain is a derived, git-ignored index — always rebuildable.

Markdown Vault (git)
  raw/             ← immutable captured sources (type: source)
  notes/
    facts/         ← atomic claims (type: fact)
    entities/      ← people, systems, concepts (type: entity)
    decisions/     ← ADRs (type: decision)
    moc/           ← maps of content (type: moc)
    questions/     ← auto-closed by dream (type: question)
  .kg/             ← derived, git-ignored
    brain.sqlite
    meta.json
    metrics.log    ← per-command token JSONL

Retrieval Pipeline
  Query
   ├─► Vector KNN (sqlite-vec, 384-dim all-MiniLM-L6-v2)
   ├─► BM25 / FTS5 (porter-stemmed keyword)
   ├─► Entity NER → 1-hop graph expansion
   └─► Global? → Leiden community summaries (GraphRAG)
         ↓ Reciprocal Rank Fusion (k=60)
   Hybrid ranked context
         ↓
   kg ask / kg recall / kg search / MCP tools

Installation

Build from source. Requires Rust 1.78+. The binary is statically linked — no runtime deps.

Build from Source

# Requires Rust 1.78+
git clone https://github.com/yourusername/kgx
cd kgx
cargo build --release        # ~30s cold build
cp target/release/kg ~/.local/bin/kg
kg --version                 # kg 0.1.0  (12.5 MB)

Initialize a Vault

mkdir ~/brain && cd ~/brain

# Research vault
kg init --template research

# Engineering team (ADRs, decisions, entities)
kg init --template code

# With AI tool skill files
kg init --template research --with-skills --okf

This scaffolds:

brain/
├── CLAUDE.md          ← agent behavior contract (read by all AI tools)
├── index.md           ← OKF root map-of-maps
├── log.md             ← append-only operation log
├── raw/               ← immutable captured sources
└── notes/
    ├── facts/
    ├── entities/
    ├── decisions/
    ├── moc/
    └── questions/

Important: All kg commands use the current working directory as the vault root. There is no --vault flag — always cd into your vault first.

Skills Setup

Skills teach each AI tool how to behave in a KGX vault. One command installs context files for all tools:

cd ~/brain
kg init --with-skills

CLAUDE.md — Agent Contract

Placed in vault root. Claude Code reads at session start and follows for every kg command or MCP tool call.

  • Note type schema and required frontmatter fields
  • Ponytail extraction ladders (lite / full / ultra intensity)
  • Dreaming rules (supersede vs merge thresholds)
  • [[wikilink]] conventions and typed link predicates
  • MOC maintenance guidelines
  • Review checklist for kg review --ponytail-audit

Example session flow

# User: "What database does the billing service use?"

# Claude Code calls MCP:
ask_question("billing service database")
→ "Based on the notes, Postgres is the primary datastore."
  citations: ["01FACT01POSTGRESPRIMARY00"]

# Then graph traversal:
kg recall --entity "Billing Service"
→ 12 neighbors including ADRs, migration notes, dependency facts

# Synthesizes with note IDs as citations

The smoke test native_skill_packages_reference_same_mcp_tools verifies that Claude, Codex, and Cursor skill files all reference the same 6 MCP tool names — preventing integration drift.

AGENTS.md — Codex Skill

Copy skills/codex/AGENTS.md to your vault root. Codex picks it up automatically.

# AGENTS.md (vault root)
Use the kg CLI and kgx MCP server when in a KGX vault.

Commands:
- kg capture --from - --type doc
- kg extract --source <NOTE_ID> --intensity full
- kg ask "<question>" --cite
- kg dream --max-iterations 3
- kg review --approve all
- kg index --full

Rules:
- raw/ is immutable — never edit after capture.
- Supersede or archive; never delete notes.
- Cite note IDs in every answer.
- type: must be one of: fact|entity|decision|experience|moc|source|question

Cursor — MCP + CLAUDE.md

Cursor uses .cursor/mcp.json for tool access and reads CLAUDE.md from the project root for behavioral context. No separate skill file needed.

# .cursor/mcp.json (vault root)
{
  "mcpServers": {
    "kgx": {
      "command": "kg",
      "args": ["mcp-server", "--transport", "stdio"]
    }
  }
}

Cursor auto-launches the MCP server on first tool call. The agent can then call search_notes, ask_question, upsert_note etc. from within the editor.
