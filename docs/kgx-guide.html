Installation

Three ways to install. No runtime dependencies — the binary is statically linked.

Option A: Release Install (recommended)

curl -fsSL https://raw.githubusercontent.com/thanhNt16/kgx/main/install.sh | bash

# Install a specific version
curl -fsSL https://raw.githubusercontent.com/thanhNt16/kgx/main/install.sh | KGX_VERSION=v0.1.0 bash

Installs kg to ~/.local/bin plus skill templates to ~/.kgx/skills.

Option B: Build from Source

git clone https://github.com/thanhNt16/kgx
cd kgx
cargo build --release        # ~30s cold, 12.5 MB binary
cp target/release/kg ~/.local/bin/kg
kg --version

Option C: Dev Install (build + wire agent)

One command builds, installs, scaffolds a vault, and wires MCP + skills for your AI tool:

./dev-install.sh                          # default: Claude Code
./dev-install.sh --agent opencode         # OpenCode
./dev-install.sh --agent codex            # Codex CLI
./dev-install.sh --agent cursor           # Cursor
./dev-install.sh --agent cursor --vault ~/my-project

1

/kgx:capture

Save source material immutably. SHA-256 dedup means the same content won't create duplicates.

/kgx:capture
→ Paste this RFC summary:
     "We decided to use Postgres as the primary datastore.
      CockroachDB is being evaluated for geo-distributed regions.
      The billing service depends on Postgres."

→ Saved to raw/2026-07-03-rfc-datastore.md
  Source note: notes/sources/rfc-datastore.md (01RAW01...)

Also supports --from file or stdin pipe. Raw files are never edited after capture — guarantees provenance.

2

/kgx:extract --source <ID>

Extract atomic facts, entities, and decisions from a captured source. Each note carries source: + recorded_at: provenance.

/kgx:extract --source 01RAW01...
→ Created 3 notes:
  • notes/facts/f-postgres-primary.md
    (Postgres is the primary datastore)
  • notes/entities/postgres.md
    (Postgres — type: entity, infra/datastore)
  • notes/entities/billing-service.md
    (Billing Service — type: entity, billing)

Also supports --batch (all unprocessed sources) and --intensity lite|full|ultra.

3

/kgx:index

Build the SQLite brain — vector search (384-dim), BM25/FTS5, PageRank, Leiden communities:

/kgx:index
→ Brain built: 17 nodes, 28 edges, 17 embedded (23ms)
  Leiden community detection found 3+ topic clusters
  Deterministic — safe to run anytime
4

/kgx:ask "question"

Hybrid vector + keyword + graph retrieval. Answers include cited note IDs.

/kgx:ask "What database does the billing service use?"
→ Based on the vault, Postgres is the primary datastore.
  [Source: Postgres is the primary datastore (01FACT01POSTGRESPRIMARY00)]

/kgx:ask "Find all decisions about datastores" --scope global
→ 3 decisions found:
  • ADR-001: Postgres as primary datastore
  • ADR-002: Migrate to CockroachDB
  • ADR-003: Database backup policy

Token efficiency: 371 input tokens vs ~3,400 pasting all 17 notes — 89% reduction.

5

/kgx:recall --entity "Name"

/kgx:recall --entity "Postgres"
→ Postgres connects to 12 neighbors:
  • Decisions: ADR-001, ADR-002
  • Entities: CockroachDB, Billing Service
  • Facts: primary datastore, backup policy
  • MOC: Datastore MOC
  • Questions: sync strategy during migration
  • Sources: Architecture Review, Migration Note
  12 neighbors in 1ms — pure graph lookup, no LLM call.

Surfaces notes from 2 sprints ago — knowledge grep misses because you wouldn't know the keywords.

/kgx:dream

→ Consolidation complete:
  • 14 diffs applied (dedup, stale archival, link repair)
  • 6 contradictions flagged for manual review
  • 2ms elapsed, no LLM call

Use Cases

Real scenarios where KGX replaces manual wiki maintenance, context-pasting, and scattered notes.

🧠

Personal Knowledge Base

"I read 20 articles and want to ask questions across all of them." Capture each → extract → index → ask across the entire corpus. Hybrid search fuses vector, keyword, and graph connectivity.

👥

Engineering Team Wiki

"What did we decide about auth? Who depends on billing?" Engineers write ADRs and entities as Markdown. kg recall --entity "Auth Service" surfaces all decisions, dependencies, and open questions.

🤖

AI Agent Long-Term Memory

"What auth decisions did we make yesterday?" — Claude Code calls nl_query_memory and retrieves from the persistent brain. No context window resets, no re-pasting notes. Agents also call ingest_conversation to capture new knowledge during chat.

🚀

Sprint Knowledge Retention

"Lost Sprint-7 context by Sprint-9." KGX surfaces Sprint-7 facts automatically via kg recall at Sprint 9. Verified: 6/6 Sprint-7 facts accessible at Sprint 9 without knowing what to search for.

📦

Portable OKF Bundles

Share project context across machines or teammates. kg ship → kg pull → kg validate --okf. Lossless round-trip. Works offline.

🌙

Unattended Wiki Gardening

Schedule kg dream via kg cron enable dream-nightly. Dedup, contradiction detection, stale archival — runs automatically. No humans required.

Architecture

Markdown is canonical. The SQLite brain is a derived, git-ignored index — always rebuildable. 18 crates in a Rust workspace.

Markdown Vault (git canonical)
  raw/             ← immutable captured sources
  notes/
    facts/         ← atomic claims (type: fact)
    entities/      ← people, systems, concepts (type: entity)
    decisions/     ← ADRs (type: decision)
    moc/           ← maps of content (type: moc)
    questions/     ← open gaps (type: question)
  .kg/             ← derived, git-ignored
    brain.sqlite   ← nodes, edges, FTS5, vectors, PPR, communities
    meta.json      ← last-run timestamps
    metrics.log    ← per-command token JSONL

Brain Layers
  Vector KNN (sqlite-vec, 384-dim MiniLM)
  BM25 / FTS5 (porter-stemmed)
  Tags-in-LIKE substring search
  Tag-frequency-weighted expansion
  Entity NER → 1-hop graph expansion
  Leiden community summaries (GraphRAG)
  Personalized PageRank (HippoRAG)
        ↓ Reciprocal Rank Fusion
  Hybrid ranked results
        ↓
  kg ask / kg recall / kg search / MCP tools
